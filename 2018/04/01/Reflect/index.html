JAVA反射机制(框架底层原理)

Class类
1.在面向对象的世界里，万事万物皆对象。
java语言中，静态的成员(属于某个类的)，普通数据类型类不是对象，是属于某个类(包装类(自动装箱))的
类是对象，类是java.lang.Class类的实例对象。 There is a class named Class
Class类的实例是每一个对象所属的类。
2.任何一个类都是Class类的实例对象，这个实例对象有三种表示方式
Foo foo1 = new Foo();
第一种—>实际告诉我们任何一个类都有一个隐含的静态成员变量：Class c1 = Foo.class;
第二种—>已知该类的对象，通过getClass()方法：Class c2 = foo1.getClass();
官网中c1,c2表示为Foo类的类类型(class type)；类也是对象，是Class类的实例对象，这个对象我们称为该类的类类型
无论c1或c2都代表了Foo类的类类型，一个类只可能是Class类的一个实例对象。c1==c2
第三种—>通过Class.forName(完整类名)：Class c3 = null; c3 = Class.forName(“包名.类名”); //捕获ClassNotFoundException
无论几种表达方式，都代表了同一个类的类类型。 c2==c3
3.完全可以通过类的类类型来创建该类的对象实例(通过c1,c2,c3创建Foo类的实例)
Foo foo = (Foo)c1.newInstance(); //捕获InstantiationException和illegalAccessException
foo.成员方法 //Foo类必须有无参数的构造方法
4.基本的数据类型，void等关键字都存在类类型
5.Class类的基本API操作 (在任何情况下想要获得类的信息，首先要获得该类的类类型，再通过类类型的各种get访问器获得该类的信息)

方法的反射
(1) 如何获取某个方法：方法的名称和方法的参数列表才能唯一决定某个方法
(2) 方法反射的操作：method.invoke(对象, 参数列表)

成员变量的反射
1.成员变量(实例域)也是对象
2.java.lang.reflect.Field
3.Field类封装了关于成员变量的操作
4.getFields()方法获取的是所有的public的成员变量的信息
5.getDeclaredFields获取的是该类自己声明的成员变量的信息(不问访问权限，private的也可获取)

构造函数的反射
1.构造函数也是对象
2.java.lang.Constructor中封装了构造函数的信息
3.getConstructors获取所有的public的构造函数
4.getDeclaredConstructors得到所有的构造方法(构造方法都是自己声明的)
5.获取构造函数的参数列表—>得到的是参数列表的类型的类类型

Java类加载机制
1.Class.forName(“类的全称”)：不仅表示了类的类类型，还代表了动态加载类；注意区分编译，运行；编译时刻加载类是静态加载类，运行时刻加载类是动态加载类。
2.new 创建对象是静态加载类，在编译时刻就需要加载所有的可能使用到的类，通过动态加载类可以避免这种情况(在运行时刻加载)。
3.通过类类型，创建该类的对象。
4.定义公共接口，实现该接口来实现动态加载类的功能。
(通过反射机制了解集合泛型的本质；通过Class, Method来认识泛型的本质)
