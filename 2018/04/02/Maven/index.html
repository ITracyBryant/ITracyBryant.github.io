Maven

Apr 2, 2018 | note | 阅读
条评论

1. Maven

  • Maven是基于项目对象模型(POM),可以通过一小段描述信息来管理项目的构建，报告和文档的软件项目管理工具。
  • bin目录包含mvn运行脚本
  • boot目录包含一个类加载器框架
  • 目录结构：
    • src
      • main
        • java
          • package
        • resources
      • test
        • java
          • package
        • resources
  1. groupId的值是项目包名
  2. artifactId的值是模块名(项目名)
  3. dependencies dependency 导入依赖
  4. 默认生成target目录
  5. mvn package生成jar包

2. Maven常用命令

  • mvn -v 查看maven版本
  • compile 编译 test测试 package打包
  • clean 删除target目录
  • install 安装jar包到本地仓库
  • archetype插件用于创建符合maven规定的目录骨架 (自动创建目录骨架)
    • mvn archetype:generate (按照提示选择)
    • mvn archetype:generate -DgroupId=项目名 -DartifactId=项目名-模块名 -Dversion=1.0.0SNAPSHOT -Dpackage=完整包名

2.1 坐标和仓库

  • 构件通过坐标作为其唯一标识
  • 仓库管理项目依赖，本地仓库和远程仓库(pom-4.0.0中央仓库)
  • 镜像仓库（在settings.xml中修改镜像仓库位置 国内 maven.net.cn）
  • 更改仓库位置 localRepository标签(本地位置)

2.2 maven的生命周期和插件

Eclipse4.0以上版本自带maven插件

  1. clean，compile，test，package，install
  2. 完整的项目构建过程：清理，编译，测试，打包，集成测试，验证，部署
  3. maven生命周期：clean 清理项目；default 构建项目；site 生成项目站点 (三套生命周期独立存在，其内部的各个阶段依次执行)
  • clean：
    • pre-clean 执行清理前的工作
    • clean 清理上一次构建生成的所有文件
    • post-clean 执行清理后的文件
  • default(最核心)：compile test package install
  • site：
    • pre-site在生成项目站点前要完成的工作
    • site 生成项目的站点文档
    • post-site 在生成项目站点后要完成的工作
    • site-deploy 发布生成的站点到服务器上

2.3 依赖范围

  • 三种classpath：编译，测试，运行
  • Dependency Scope：
    • compile：默认的范围，编译测试运行都有效
    • provided：在编译和测试时有效(Servlet)
    • runtime：在测试和运行时有效(JDBC)
    • test：只在测试时有效(JUnit)
    • system：编译和测试时有效，与本机系统关联，可移植性差
    • import：导入的范围，它只使用在dependencyManagement中，表示从其他的pom中导入dependency的配置(继承过来的依赖)

2.4 依赖传递和依赖冲突

  • 短路优先(优先解析路径短的版本)
    • A -> B -> C -> X（jar）
    • A -> D -> X(jar) 优先解析该版本
  • 先声明先优先(如果路径相同，则谁先声明，先解析谁)

2.5 聚合和继承

  • 聚合：modules标签
  • 继承：父pom中使用dependencyManagement标签，子pom中使用parent标签继承父pom的坐标
