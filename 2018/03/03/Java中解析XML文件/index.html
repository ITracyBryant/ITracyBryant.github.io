Java中解析XML文件

Mar 3, 2018 | note | 阅读

基础方法：DOM(与平台无关的官方解析XML方式)，SAX(JAVA平台提供的基于事件驱动的解析方式)
扩展方法：JDOM，DOM4J(在基础方法上扩展出的，只有Java中能够使用的解析方式),基于底层API

DOM解析：一次性将xml文件内容加载进内存中，形成一个DOM树
SAX解析：基于事件的解析，通过自定义创建的Handler处理类逐个解析每一个节点，
解析过程按顺序(由外向内依次)进行。startDocument(),startElement(),endElement(),endDocument()

DOM优势：形成DOM树，直观便于理解，代码易于编写；解析过程中树结构保留在内存中，方便修改。
DOM劣势：当xml文件较大时，对内存耗费较大，容易影响解析性能并造成内存溢出
SAX优势：采用事件驱动模式，对内存耗费较小；适用于只需要处理xml中数据时。
SAX劣势：不易编码；很难同时访问同一个xml中的多处不同数据

JDOM解析：仅使用具体类而不使用接口；API中大量使用Collections类；开源
DOM4J解析: JDOM的智能分支，合并了许多超出基本xml文档表示的功能(对额外的深层次的xml功能支持更完善)；
DOM4J使用接口和抽象基本类方法，是一个优秀的Java xml API；
性能优异，灵活性好，功能强大,易使用；开源 (nice)

生成XML的比较：
生成XML文件时，DOM树驻留在内存中，能够随时删除，修改，重新排列节点
SAX基于事件的，无状态性，顺序执行，无法修改，增加或删除；性能更高（最快）
