JSP指令与动作&JSP中Cookie的使用

Mar 30, 2018 | note | 阅读

JSP指令与动作
include指令：

include动作：(动作标签)
page：要包含的页面 flush：被包含的页面是否从缓冲区读取(默认false)
(典型面试问题)两者区别：
1.语法格式
2.发生作用的时间：include指令–>页面转换期间；include动作–>请求期间
3.包含的内容：include指令–>文件的实际内容(代码)；include动作–>页面的输出(结果)
4.转换成的servlet：include指令–>主页面和包含页面转换成一个Servlet；include动作–>主页面和包含页面转换为独立的Servlet
5.编译时间：include指令–>较慢(资源必须被解析)；include动作–>较快
6.执行时间：include指令–>稍快；include动作–>较慢(每次资源必须被解析) (include动作在请求期间被执行，include指令在编译期间被执行)
<页面内容经常变化适合使用include动作，不经常变化适合使用include指令>
forward动作：等同于request.getRequestDispatcher("/url").forward(request,response) param动作：常常与forward动作一起使用，作为其的子标签
plugin动作：标签表示执行一个applet或Bean(可能要下载浏览器Java插件执行)

JSP中Cookie的使用
JSP状态管理
1.HTTP协议无状态性：当浏览器发送请求给服务器时，服务器响应客户端请求，但当同一个浏览器再次发送请求给服务器时，服务器并不知道它就是刚刚的客端浏览器；服务器不保存 用户的状态
2.保存用户状态的两大机制：Session，Cookie
Cookie：注册表，是Web服务器保存在客户端浏览器的一系列文本信息(字符串)
典型应用：(1)判定注册用户是否已经登录网站
(2)购物车的处理(保存用户浏览商品的记录);保存浏览记录和登录状态
作用：对特定对象的追踪，保存用户网页浏览记录与习惯，简化登录
安全风险：泄露隐私风险
创建和使用Cookie：
(1)创建Cookie对象：Cookie newCookie = new Cookie(String key, Object value);
(2)写入Cookie对象：response.addCookie(newCookie);
(3)读取Cookie对象：Cookie[] cookies = request.getCookies();
常用API：
void setMaxAge(int expiry) 设置Cookie的有效期，以秒为单位
void setValue(String value) 在Cookie创建后，对Cookie进行赋值
String getName() 获取Cookie的名称
String getValue() 获取Cookie的值
int getMaxAge() 获取Cookie的有效时间，以秒为单位
3.Session与Cookie比较：
session：在服务器端保存用户信息；保存的是Object类型对象(没有限制)；随会话的结束而将其存储的数据销毁；保存重要信息安全性较高
cookie：在客户端保存用户信息；保存的是String类型(对保存对象大小有限制)；可以长期保存在客户端；一般保存的是不重要的用户信息安全性较低
