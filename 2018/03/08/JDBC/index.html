JDBC

Mar 8, 2018 | note | 阅读

JDBC(Java DataBase Connectivity)java数据库连接，可以为多种数据库提供统一的访问，体现了Java“一次编写，处处运行”的思想(连接数据库的中间件)
JDBC编程步骤：
1.加载驱动程序：Class.forName(driverClass)
加载Mysql驱动：Class.forName(“com.mysql.jdbc.Driver”);
加载Oracle驱动：Class.forName(“oracle.jdbc.driver.OracleDriver”)
2.获得数据库连接：
DriverManager.getConnection(“jdbc:mysql://127.0.0.1:3306/test”,”root”,”1003”);
3.创建Statement对象：conn.createStatement();

JDBC实战(实际开发详设)：
采用MVC三层架构：View-Control-Model–DB —》(架构模型从上往下)，实际开发流程最好从下往上
Model(对应DB的映射，对应DB映射的抽象方法(增删改查CRUD)，即数据和业务逻辑的处理，然后通知Controller)
Controller(控制数据的流通过程，控制CRUD方法的运用，更新Model，最后将数据拼装起来传递给View)
View(对数据的展示，将数据显示到前台，将用户操作的数据传递给Controller)
C:create(新增)，R:query(查询：简单查询，高级查询(报表，分页等))，U:update(更新)，D:delete(删除)
View层流程：
1.程序启动后，一直保持在运行状态；
2.循环接收控制台(view层)的用户输入；
3.调用Action(Controller)响应，并将返回结果展示在控制台(View)；
4.直到监听到用户输入了特定标记(如exit)后，程序退出
keypoint:1.循环接收参数；2.某个特定功能的保持(保持数据的正确操作)

JDBC各种连接方式的对比：
1.JDBC+ODBC桥的方式
特点：需要数据库的ODBC驱动，仅适用于微软的系统(JDBC调用传递给ODBC，ODBC再调用本地的数据库驱动)
2.JDBC+厂商API的形式
特点：厂商API一般用C编写
3.JDBC+厂商DataBase Connection Server + DataBase的形式
特点：在JAVA与DATABASE之间架起一台专门用于与数据库连接的服务器(一般由数据库厂商提供)
4.JDBC+DataBase的连接方式(常用，小型应用)
特点：这使Application与数据库分离，应用程序开发者只需关心内部逻辑实现而不用注重数据库连接的具体实现(connection被封装)
